# AGENTS

このリポジトリに関連するエージェントガイドです。各エージェントは iOS アプリ開発の役割に沿って補完的に機能します。

## エージェント一覧

- `ProductAgent`
  - プロダクト要件、ユーザー体験、MVP の定義を支援します。
  - 仕様書、UI ストーリー、優先順位付けを整理します。

- `DesignAgent`
  - 画面遷移、UI 要素、ユーザー操作フローの設計を支援します。
  - UI コンポーネントとアクセシビリティの整合性を確認します。

- `ArchitectureAgent`
  - iOS 固有のアーキテクチャ、モジュール構成、依存関係を設計します。
  - SwiftUI、サービス層、データストレージの責務分割を明確化します。

- `ImplementationAgent`
  - コード実装、タスク分割、実装プランの作成を支援します。
  - 既存コードに対する変更案とテスト戦略を提示します。

- `ReviewAgent`
  - 仕様書と実装の整合性をレビューします。
  - ドキュメントの追跡、設計決定のレビュー、改善提案を行います。

## 利用方法

1. まず `docs/PRODUCT_SPEC.md` と `docs/UI_SPEC.md` を確認し、アプリの目的と主要画面を把握します。
2. `docs/IOS_ARCHITECTURE.md` でアーキテクチャ構造を確認します。
3. 実装中は `docs/IMPLEMENTATION_PLAN.md` に沿ってタスクを進めます。
4. 変更点や決定事項は `docs/DECISIONS.md` に逐次追記します。

## 注意事項

- このプロジェクトはクラウド連携を含めないローカルファースト設計です。
- API 仕様は既存の内部サービス（ローカル JSON、Photo Library）にフォーカスします。
- ドキュメントの変更は `docs/` 以下に統一して管理します。

# DECISIONS

## 1. ローカルファースト設計

- すべてのデータは端末内に保存し、クラウド同期は行わない。
- プライバシーを重視し、外部通信を不要とする方針。

## 2. 写真は写真ライブラリに保存

- 画像データはアプリ内ストレージではなく、iOS 写真ライブラリに保存する。
- `localIdentifier` を使って画像参照を管理することで、ストレージ容量や PhotoKit の利便性を活かす。

## 3. JSON ファイルを永続化に採用

- `LabelStore` で `Application Support` に JSON ファイル保存を採用。
- データベース導入は不要なため、シンプルな構造を優先。

## 4. SwiftUI を UI フレームワークに選択

- 既存のコードベースが SwiftUI で構築されているため、継続して SwiftUI を使用。
- `NavigationStack` / `sheet` を使い、画面遷移をシンプルに保つ。

## 5. サービス層を明確に分離

- `LabelStore` と `PhotoLibraryService` を責務別に分離する。
- Views はデータ操作を直接行わず、サービス経由で操作する。

## 6. テスト戦略は徐々に追加

- まずは動作確認と QA を優先し、必要に応じてユニットテストを追加する。
- 画像保存などの外部依存部分はモック化してテスト可能にする。

## 7. 新規追加は `+` から統一

- 一覧の右上 `+` から新規追加を開始する導線に統一する。
- 画像追加は編集画面内で `カメラ` と `フォトライブラリ` の両方を選べるようにする。

## 8. 詳細画面は参照専用 + 明示的編集

- 詳細画面は表示専用とし、`編集` ボタンで編集画面に遷移する。
- 削除は詳細画面で確認ダイアログを表示してから確定する。

## 9. カメラ撮影画像はメタデータ保持で保存

- カメラ撮影した画像は `UIImage` の再エンコード時に EXIF / TIFF メタデータを埋め戻して写真ライブラリへ保存する。
- フォトライブラリから選択した既存写真は再保存せず、既存 `PHAsset` の `localIdentifier` をそのまま参照する。
- 保存したメタデータは UI へ表示せず、写真ライブラリ上の原本保持を優先する。

## 画面構成

### 1. 起動スプラッシュ画面

- アプリ起動直後に表示する
- 設定済みのアプリアイコン画像とアプリ名を中央表示する
- 短時間表示した後、自動でラベル一覧画面へ遷移する
- ユーザー操作は要求しない

### 2. ラベル一覧画面

- ナビゲーションバー中央: ボトルアイコン + `サケラベル`
- 左上ボタン: ハンバーガーメニュー
- 右上ボタン: `+`（新規追加）
- ナビゲーションバーはインライン表示とし、ヘッダー下に大タイトル用の余白を残さない
- ハンバーガーメニュー選択時は、左側からスライドインするドロワー形式でメニューを表示する
- 上部フィルター: `すべて` `日本酒` `ワイン` `ウイスキー`
- ハンバーガーメニュー項目:
  - `設定`
  - `その他`
- セル項目:
  - ラベル画像のサムネイル
  - ラベル名
  - お酒ラベル
  - レーティング
  - メモの先頭数行
  - 登録日時
- セルタップで詳細画面へ遷移
- セル長押しまたはコンテキストメニューから削除操作を開始できる
- フィルター選択時は、該当ラベルのエントリのみ一覧表示する
- データは `登録日時` の降順で並ぶ
- 空状態:
  - 登録データが 0 件のときは新規追加を促す空状態を表示する
  - 絞り込み結果が 0 件のときは該当ラベルがないことを示す空状態を表示する

### 2-1. 設定画面

- 表示項目:
  - アプリバージョン（`CFBundleShortVersionString`、例: `1.0.0`）
- 一覧画面のハンバーガーメニュー `設定` から開く
- モーダル表示し、右上 `閉じる` で閉じる
- 画面背景色は一覧画面と同じ背景色を使用する

### 2-2. その他画面

- 一覧画面のハンバーガーメニュー `その他` から開く
- 表示項目:
  - `利用規約`
  - `プライバシーポリシー`
- それぞれ専用の文書表示画面へ遷移できる
- モーダル表示し、右上 `閉じる` で閉じる
- 画面背景色は一覧画面と同じ背景色を使用する

### 2-3. 利用規約／プライバシーポリシー画面

- スクロール可能な静的文書画面
- セクションごとにカード表示する
- `その他` 画面から内部遷移で表示する

### 3. ラベル詳細画面

- 表示項目:
  - ラベル画像（大きめ）
  - ラベル名
  - お酒ラベル
  - レーティング
  - 詳細メモ
  - 登録日時
- レイアウト要件:
  - ラベル名、詳細メモ、登録日時の各カードは同一幅で表示し、メモカードと同じ横幅基準に揃える
  - 各カードは文字数に応じて横幅が縮まらないこと
  - 各カード内のテキストは見出し・本文ともに左寄せで表示する
  - お酒ラベルとレーティングは一覧画面と同じチップ表現を使用する
- 編集ボタン

### 4. ラベル編集／新規追加画面

- イメージプレースホルダー
- イメージプレースホルダーまたは選択済み画像プレビューをタップすると、画像選択／撮影フローに進む
- テキストフィールド: `ラベル名`
- `ラベル名` は任意入力とし、未入力でも保存可能とする
- レーティング入力: 5段階の星で選択可能、未評価も許容
- お酒ラベル選択: `日本酒` `ワイン` `ウイスキー` のプリセットから 1 つ選択
- お酒ラベルの選択 UI は横並びではなく、ロールを回転させるホイール形式とする
- テキストエリア: `メモ`
- 保存ボタン
- キャンセルボタン（モーダルの場合）
- 編集モードでは画面下部に削除ボタンを表示し、削除確認ダイアログを経て削除を確定できる

### 5. 画像選択／撮影画面

- iOS 標準の `UIImagePickerController`
- ユーザーが写真を選択またはカメラ撮影を実行
- 選択後、自動で保存画面に戻る
- カメラ撮影した画像は `9:16`、`2268 x 4032` の保存形式で扱う
- カメラ撮影から登録した場合、登録日時には撮影時刻を採用する
- フォトライブラリから選択した画像は、画像に紐づく EXIF / メタ情報を保持したまま登録処理へ引き渡す
- フォトライブラリから登録した場合、登録日時には EXIF 上の撮影時刻を採用する
- カメラ撮影時の `Cancel` `Photo` などの標準文言は日本語ローカライズされた表示とする
- カメラ撮影画面は上下の補助領域を黒基調で表示し、白く浮かないこと
- カメラプレビューは無理に拡大せず、iPhone 標準写真と同じ自然なアスペクト比で表示する
- 写真アプリ側の元画像が削除された場合でも、アプリ内バックアップ画像でサムネイルと詳細表示を継続できること
- バックアップ画像は縮小版とし、長辺 `960px` を上限とする
- カメラ撮影時は `UIImagePickerController` をフルスクリーンで表示する
- フォトライブラリ選択時も登録処理用のアプリ内バックアップ画像を作成する

## 操作フロー

- アプリ起動 → 起動スプラッシュ画面表示 → ラベル一覧画面表示
- 一覧画面左上メニュー → 設定 → アプリバージョン確認
- 一覧画面左上メニュー → その他 → 利用規約 / プライバシーポリシー確認
- 一覧カード長押し → コンテキストメニュー → 削除確認
- `+` タップ → 編集画面表示 → 画像追加 → ラベル名（任意）／メモ入力 → 保存
- `+` タップ → 編集画面表示 → 画像追加 → ラベル名（任意）／レーティング／お酒ラベル／メモ入力 → 保存
- 一覧セルタップ → 詳細画面表示
- 詳細画面編集 → 編集画面表示 → 更新保存 または 削除

## UI/UX ポイント

- シンプルで明快な一覧表示
- 撮影画像が配信される形で一覧に見える
- 端末内保存の安心感を前面に出す
- 編集画面はフォームを縦に並べる
- キーボード表示時にスクロールする
- 詳細画面の情報カードは幅と文字揃えを統一し、情報量が少ない項目でも間延びせず整った見た目を維持する

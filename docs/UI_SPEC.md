# UI_SPEC

## 画面一覧

1. ラベル一覧画面
2. ラベル詳細画面
3. ラベル編集／新規追加画面
4. 画像選択／撮影画面（システムの ImagePicker）

## 画面構成

### 1. ラベル一覧画面

- ナビゲーションバータイトル: `Sake Label Notes`
- 右上ボタン: `+`（新規追加）
- セル項目:
  - ラベル画像のサムネイル
  - ラベル名
  - メモの先頭数行
  - 追加日時または更新日時
- セルタップで詳細画面へ遷移

### 2. ラベル詳細画面

- 表示項目:
  - ラベル画像（大きめ）
  - ラベル名
  - 詳細メモ
  - 保存日時／更新日時
- 編集ボタン
- 削除ボタン

### 3. ラベル編集／新規追加画面

- イメージプレースホルダー
- `写真を追加` ボタン
- テキストフィールド: `ラベル名`
- テキストエリア: `メモ`
- 保存ボタン
- キャンセルボタン（モーダルの場合）

### 4. 画像選択／撮影画面

- iOS 標準の `PHPickerViewController` または `UIImagePickerController`
- ユーザーが写真を選択またはカメラ撮影を実行
- 選択後、自動で保存画面に戻る

## 操作フロー

- `+` タップ → 編集画面表示 → 画像追加 → ラベル名／メモ入力 → 保存
- 一覧セルタップ → 詳細画面表示
- 詳細画面編集 → 編集画面表示 → 更新保存
- 詳細画面削除 → 削除確認ダイアログ → 一覧へ戻る

## UI/UX ポイント

- シンプルで明快な一覧表示
- 撮影画像が配信される形で一覧に見える
- 端末内保存の安心感を前面に出す
- 編集画面はフォームを縦に並べる
- キーボード表示時にスクロールする

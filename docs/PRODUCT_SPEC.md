# PRODUCT_SPEC

## プロダクト概要

SakeLabelNotes は、ユーザーが日本酒やワインなどの瓶ラベルの写真とメモを組み合わせて保存・管理できる iOS アプリです。すべてのデータは端末内に保存し、クラウド連携は行いません。
お酒の種類は日本酒だけでなく、ワイン、ウイスキーなど瓶のラベルを想定します。

## ターゲットユーザー

- 家飲みでお気に入りの日本酒を記録したい人
- 酒ラベル情報と感想を簡単にまとめたい人
- 写真付きメモをローカルで完結させたい人

## 主な課題

- お酒のラベルを後で見返したい
- 味や購入場所などをメモとして残したい
- 画像とテキストを一緒に保存したい

## MVP 機能

- ラベル写真を iPhone カメラで撮影
- 撮影画像を写真ライブラリに保存
- ラベル名とメモを入力してローカル JSON に保存
- 保存エントリの一覧表示
- 詳細表示、編集、削除

## 成功指標

- アプリを起動してすぐに新しいラベルを追加できる
- 保存したラベルが一覧で正しく表示される
- 画像付きメモの追加・編集・削除が安定して動作する
- 端末内に保存され、プライバシーが保たれる

## 機能概要

### 記録

- レーティングを行える
  - 5段階の「☆」をつけることで簡単に美味しかったかどうかを書き込めるようにすること
  - 未評価の状態も許容し、あとから編集で設定できること
- 詳細なメモも必須ではないが付け加えることができるようにすること
- ラベル機能。お酒の種類に応じたラベルを割り当てることができる
  - 「ワイン、ウイスキー、日本酒」をプリセットとして用意すること
  - ラベルで絞り込み検索も行うことができること。
  - 新規追加／編集時にいずれか 1 つのラベルを選択して保存できること

### 一覧表示機能

- 一覧画面では、各エントリについてラベル画像・ラベル名・メモに加えて、レーティングとお酒ラベルを確認できること
- レーティングとお酒ラベルは、詳細画面に入らなくても一覧カード上で視認できること
- 一覧画面では、お酒ラベルごとに保存済みエントリを絞り込み表示できること
- 絞り込み対象は `すべて` `日本酒` `ワイン` `ウイスキー` とし、切り替え後は該当するエントリだけを一覧に表示すること

### カメラ機能

- ユーザーはアプリを通じてカメラ撮影を行うことができる。撮影した画像はそのまま iphone 写真アプリに保存される。
  - 撮影画像は `9:16` の縦長フォーマット、`2268 x 4032` ピクセルで保存すること
  - その際に、カメラ情報の exif も保存すること。レンズ情報、撮影日などのメタ情報を持った画像を保存すること
  - 保存したメタ情報はアプリ内部で保持するが、アプリ UI 上での表示は必須としない
  - 撮影画像は写真アプリへの保存に加えて、解像度を落としたバックアップ画像をアプリ内ストレージにも保持すること
  - 写真アプリ側で元画像が削除された場合でも、アプリ内バックアップにより一覧・詳細から閲覧できること
- カメラUIは iPhone ネイティブの画面をそのまま流用すること（シートなどでアプリの上にカメラUIを表示しないこと）
- カメラ撮影画面の標準操作文言は、アプリの提供言語に合わせて自然な日本語で表示されること
- カメラ撮影画面の周辺 UI は暗色で統一し、上下の補助領域が白く見えないこと
- カメラプレビューおよび撮影画像は、iPhone 標準の写真撮影と同じ自然なアスペクト比を維持すること

### 画像登録機能（フォトライブラリからの追加）

- すでにユーザーが保存している画像を登録することができる。
  - ただしすでに写真アプリ上に保存されている場合は、アプリに登録して新規画像として作製するのではなく既存の画像の複製などは作成しないこと
  - 登録時には、選択した画像に紐づく EXIF / メタ情報も引き継いで扱うこと
  - 登録日時は、画像の EXIF 上の撮影日時を優先して採用すること
- 新規追加／編集画面では、ラベル画像エリア自体をタップすることで画像追加導線に入れること
- 画像追加用の独立したボタンは必須とせず、未設定プレースホルダーまたは選択済み画像プレビューをそのまま操作起点にすること
- フォトライブラリから選択した既存画像については、アプリ内バックアップ作成を必須としない

### 詳細閲覧画面

- ユーザーが登録した情報を閲覧するときに、ラベル名、メモ、登録日時などの情報カードは文字数に依存して横幅が縮まらないこと
- 各情報カード内のテキストボックスは画面横幅に応じて常に十分な幅を確保し、メモ表示と同じ基準の横幅に揃えること
- 各情報カード内のテキストは中央寄せにせず、見出し・本文ともに左寄せで統一すること
- お酒ラベルとレーティングはプレーンテキストではなく、一覧画面と同一デザインのチップ表現で表示すること
- 日時情報は `登録日時` のみを扱い、作成日時・更新日時を別に管理しないこと
- カメラ撮影から登録した場合、登録日時には撮影時刻を反映すること
- フォトライブラリから登録した場合、登録日時には画像の EXIF 上の撮影時刻を反映すること
